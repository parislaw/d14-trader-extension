Manual Test Steps for Violation Override Feature

Test Scenario: User marks "Followed Rules" but later violates

  1. Load Extension: Load the extension in Chrome Developer Mode
  2. Initial Check-in: Click "Followed My Rules" button
  3. Verify Completion: Confirm day is marked as completed (green) and streak increments
  4. Report Late Violation: Click "Violated a Rule" button (should still be active)
  5. Enter Violation: Fill in violation text (e.g., "Revenge traded after loss")
  6. Submit Violation: Click "Reset Streak"
  7. Verify Override:
    • Streak should reset to 0
    • Previously completed day should now show as violated (red)
    • Calendar grid should reflect the change

Expected Behavior:

  • ✅ Both buttons remain active after initial check-in
  • ✅ Violation can be reported even after marking "Followed Rules"
  • ✅ Previous completion is properly overridden to violation
  • ✅ Streak resets to 0
  • ✅ Calendar visually updates to show violation
  • ✅ Violation text is logged for accountability

Load Extension Instructions:

  1. Open Chrome and go to chrome://extensions/
  2. Enable "Developer mode" (toggle in top-right)
  3. Click "Load unpacked"
  4. Select the prop-trader-extension14days/ directory
  5. Click on the extension icon in Chrome toolbar to test
